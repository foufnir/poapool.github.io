10. Démarrage de notre stake pool

Le grand jour est arrivé!!!

Nous allons maintenant enfin démarrer notre stake pool. Mais juste avant, nous devons changer certaines choses dans les fichiers topology.json de notre noeud producteur de blocs et de notre noeud relais.

Configuration du fichier topology.json sur votre bp-node#

Le fichier topology.json est pour un noeud, ce qu'était une carte pour Indiana Jones. Le fichier donne toutes les informations nécessaires au noeud pour qu'il puisse se connecter et échanger avec d'autres noeuds. C'est d'une importance capitale.

10.01 Connectez-vous au VPS sur lequel fonctionne votre noeud producteur de blocs.

10.02 Exécutez la commande suivante pour modifier le fichier topology.json:

nano cardano-node/bp-node/mainnet-topology.json

Vous devriez normalement voir quelque chose comme ça:

10.03 Supprimez tout et copiez-collez cela à la place:

note

Remplacez TheIPaddressOfYourRelayNodeVPS par l'adresse IP du VPS de votre noeud relais.

{
"Producers": [
{
"addr": "TheIPaddressOfYourRelayNodeVPS",
"port": 3000,
"valency": 1
}
]
}

10.04 Enregistrez le fichier, quittez l'éditeur et modifions le fichier .bashrc pour que tout cela fonctionne:

note

Pour revenir au terminal, appuyez sur ctrl + o pour enregistrer le fichier, et ctrl + x pour quitter.

nano .bashrc

10.05 Ajoutez ce qui suit à votre fichier .bashrc:

note

  • Remplacez YourUserNameOnYourVPS par le nom d'utilisateur que vous utilisez sur le VPS de votre bp-node (noeud producteur de blocs).

  • Remplacez TheIPaddressOfYourVPS par l'adresse IP du VPS de votre bp-node.

Dans notre cas, la commande ressemble à ceci:

alias bplaunch="cardano-node run --topology /home/foufnir/cardano-node/bp-node/mainnet-topology.json --database-path /home/foufnir/cardano-node/bp-node/db --socket-path /home/foufnir/cardano-node/bp-node/db/node.socket --host-addr I'm-Not-Giving-You-The-IP --port 3000 --config /home/foufnir/cardano-node/bp-node/mainnet-config.json --shelley-kes-key /home/foufnir/cardano-node/pool-keys/kes.skey --shelley-vrf-key /home/foufnir/cardano-node/pool-keys/vrf.skey --shelley-operational-certificate /home/foufnir/cardano-node/pool-keys/node.cert"
alias bplaunch="cardano-node run --topology /home/YourUserNameOnYourVPS/cardano-node/bp-node/mainnet-topology.json --database-path /home/YourUserNameOnYourVPS/cardano-node/bp-node/db --socket-path /home/YourUserNameOnYourVPS/cardano-node/bp-node/db/node.socket --host-addr TheIPaddressOfYourVPS --port 3000 --config /home/YourUserNameOnYourVPS/cardano-node/bp-node/mainnet-config.json --shelley-kes-key /home/YourUserNameOnYourVPS/cardano-node/pool-keys/kes.skey --shelley-vrf-key /home/YourUserNameOnYourVPS/cardano-node/pool-keys/vrf.skey --shelley-operational-certificate /home/YourUserNameOnYourVPS/cardano-node/pool-keys/node.cert"

10.06 Enregistrez le fichier, quittez l'éditeur et actualisez le fichier .bashrc:

note

Pour revenir au terminal, appuyez sur ctrl + o pour enregistrer le fichier, et ctrl + x pour quitter.

source .bashrc

10.07 Il est temps de démarrer le bp-node! Exécutez la commande suivante:

bplaunch

Et voilà !! Vous devriez maintenant voir votre noeud producteur de blocs démarrer.

Cependant, nous devons encore configurer le noeud relais pour qu'il interagisse avec le bp-node.

Configuration du fichier topology.json de votre relay-node#

10.08 Connectez-vous au VPS de votre noeud relais.

10.09 Exécutez la commande suivante pour modifier le fichier topology.json:

note

Comme vous avez utilisé le même guide pour créer le relay-node, vous avez probablement appelé le dossier du relais bp-node. Si vous le souhaitez, vous pouvez modifier son nom pour plus de clarté. Vous devrez néanmoins modifier à chaque fois le nom du dossier dans les commandes suivantes.

nano cardano-node/bp-node/mainnet-topology.json

Vous devriez normalement voir quelque chose comme ça:

10.10 Supprimez tout et copiez-collez plutôt ceci:

note

Remplacez TheIPaddressOfYourBlockProducingNodeVPS par l'adresse IP du VPS de votre noeud producteur de blocs.

{
"Producers": [
{
"addr": "relays-new.cardano-mainnet.iohk.io",
"port": 3001,
"valency": 2
},
{
"addr": "TheIPaddressOfYourBlockProducingNodeVPS",
"port": 3000,
"valency": 1
}
]
}

10.11 Enregistrez le fichier, quittez l'éditeur et modifions le fichier .bashrc pour que tout cela fonctionne:

note

Pour revenir au terminal, appuyez sur ctrl + o pour enregistrer le fichier, et ctrl + x pour quitter.

nano .bashrc

10.12 Ajoutez ce qui suit à votre fichier .bashrc:

note

  • Remplacez YourUserNameOnYourVPS par le nom d'utilisateur que vous utilisez sur le VPS de votre noeud relais.

  • Remplacez TheIPaddressOfYourVPS par l'adresse IP du VPS de votre noeud relais.

Dans notre cas, la commande ressemble à ceci:

alias relaylaunch="cardano-node run --topology /home/foufnir/cardano-node/bp-node/mainnet-topology.json --database-path /home/foufnir/cardano-node/bp-node/db --socket-path /home/foufnir/cardano-node/bp-node/db/node.socket --host-addr I'm-Not-Giving-You-The-IP --port 3000 --config /home/foufnir/cardano-node/bp-node/mainnet-config.json"
alias relaylaunch="cardano-node run --topology /home/YourUserNameOnYourVPS/cardano-node/bp-node/mainnet-topology.json --database-path /home/YourUserNameOnYourVPS/cardano-node/bp-node/db --socket-path /home/YourUserNameOnYourVPS/cardano-node/bp-node/db/node.socket --host-addr TheIPaddressOfYourVPS --port 3000 --config /home/YourUserNameOnYourVPS/cardano-node/bp-node/mainnet-config.json"

10.13 Enregistrez le fichier, quittez l'éditeur et actualisez le fichier .bashrc:

note

Pour revenir au terminal, appuyez sur ctrl + o pour enregistrer le fichier, et ctrl + x pour quitter.

source .bashrc

10.14 Pour démarrer le noeud relais, exécutez la commande suivante:

relaylaunch

Vous devriez maintenant voir votre noeud relais démarrer.

Félicitations !! Vous avez désormais une stake pool Cardano fonctionnelle !!
