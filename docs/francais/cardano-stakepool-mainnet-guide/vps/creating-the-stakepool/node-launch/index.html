5. Démarrage de notre noeud producteur de blocs

Après avoir configuré notre noeud, il est maintenant temps de le lancer. Mais d'abord, nous devrons télécharger les fichiers de configuration et apporter quelques petites modifications pour nous assurer que tout fonctionne correctement.

Téléchargement et configuration des fichiers de configuration#

5.01 Tout d'abord, créons un dossier pour notre noeud, dans le répertoire cardano-node:

cd
cd cardano-node
mkdir bp-node
cd bp-node

5.02 Vous êtes maintenant dans le dossier bp-node, que le noeud producteur de blocs utilisera. Exécutez les commandes suivantes pour télécharger les fichiers de configuration nécessaires:

wget https://hydra.iohk.io/job/Cardano/cardano-node/cardano-deployment/latest-finished/download/1/mainnet-config.json
wget https://hydra.iohk.io/job/Cardano/cardano-node/cardano-deployment/latest-finished/download/1/mainnet-byron-genesis.json
wget https://hydra.iohk.io/job/Cardano/cardano-node/cardano-deployment/latest-finished/download/1/mainnet-shelley-genesis.json
wget https://hydra.iohk.io/job/Cardano/cardano-node/cardano-deployment/latest-finished/download/1/mainnet-topology.json

5.03 Exécutez maintenant la commande ls pour vérifier si vous avez téléchargé tous les fichiers:

ls

5.03 Nous avons maintenant tous les fichiers de configuration nécessaires, mais apportons quelques modifications au fichier mainnet-config.json:

nano mainnet-config.json

5.05 Enregistrez le fichier et exécutez la commande suivante pour sortir des répertoires dans lequel vous vous trouvez:

note

Pour revenir au terminal, enregistrez le fichier avec ctrl + o, et quittez l'éditeur avec ctrl + x.

cd

Démarrage de notre premier noeud#

5.06 La commande suivante est celle que vous utiliserez pour lancer votre noeud. Avant de l'exécuter dans le terminal, assurez-vous de changer partout YourUSERNAME avec votre nom d'utilisateur sur le VPS, ainsi que de changer TheIPaddressOfYourVPS avec l'adresse IP de votre VPS.

note

Vous devrez peut-être exécuter la commande suivante avant la commande de démarrage du noeud, sinon le système ne saura pas où se trouve l'exécutable cardano-node.

source .bashrc
cardano-node run --topology /home/YourUSERNAME/cardano-node/bp-node/mainnet-topology.json --database-path /home/YourUSERNAME/cardano-node/bp-node/db --socket-path /home/YourUSERNAME/cardano-node/bp-node/db/node.socket --host-addr TheIPaddressOfYourVPS --port 3000 --config /home/YourUSERNAME/cardano-node/bp-node/mainnet-config.json

Votre noeud est maintenant en cours de synchronisation avec la blockchain. Cela va prendre du temps. Attendez que la valeur de epoch / slot sur le tableau de bord atteigne l'époque et le slot disponibles sur PoolTool. Lorsque c'est le cas, votre noeud sera entièrement synchronisé avec la blockchain.
