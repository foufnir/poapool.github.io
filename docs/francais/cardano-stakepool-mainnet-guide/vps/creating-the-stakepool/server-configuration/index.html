3. Configurer notre serveur

Dans cette troisième partie du guide, nous allons configurer notre premier serveur. Cela signifie que nous allons installer les paquets requis, créer un utilisateur "non-root", ainsi que configurer des pare-feu pour protéger notre noeud producteur de blocs.

note

À partir de maintenant, tout (sauf indication contraire) se passera sur votre VPS.

Créer un nouvel utilisateur#

Nous sommes actuellement connectés à notre serveur en tant que "root". Root est un utilisateur qui a tous les privilèges administrateur, considérez-le un peu comme le poste de dictateur. Avoir la capacité de devenir un dictateur ne devrait pas exister, eh bien, c'est exactement la même chose avec l'utilisateur root. Pourquoi? Parce que si quelqu'un est capable d'accéder à l'utilisateur root, il aura la possibilité de contrôler l'ensemble du serveur, car l'utilisateur root a tous les privilèges administrateur.

C'est la raison pour laquelle nous n'utiliserons "root" que temporairement, afin de configurer notre serveur, mais après, nous passerons à l'utilisateur que nous allons créer maintenant.

3.01 Créez un nouvel utilisateur avec la commande suivante:

note

Chaque fois que vous voyez "YourUserName", supprimez-le et remplacez-le par le nom d'utilisateur que vous souhaitez utiliser. Dans notre cas, ce sera "foufnir".

useradd YourUserName && passwd YourUserName

3.02 Maintenant, nous devons ajouter cet utilisateur dans le "sudo group", ainsi lui aussi pourra avoir des privilèges administrateur:

usermod -aG sudo YourUserName

3.03 Changeons d'autres privilèges

sudo visudo
note

Vous êtes maintenant dans l'éditeur "nano". Il s'agit d'un éditeur dans le terminal qui vous permet de créer et d'éditer des fichiers. Pour quitter l'éditeur, appuyez d'abord sur ctrl + o pour enregistrer, et ctrl + x pour quitter l'éditeur.

3.04 Maintenant, ajoutez la ligne de code suivante sous "User privilege specification":

YourUserName ALL=(ALL:ALL) ALL
note

Assurez-vous de remplacer "YourUserName" par le nom d'utilisateur que vous avez créé au début de cette partie du guide.

3.05 Exécutez maintenant les commandes suivantes dans le bon ordre et assurez-vous de toujours remplacer YourUserName par votre nom d'utilisateur:

mkdir /home/YourUserName
chown YourUserName:YourUserName /home/YourUserName -R
rsync --archive --chown=YourUserName:YourUserName ~/.ssh /home/YourUserName
chsh -s /bin/bash YourUserName

Mise à jour et installation des paquets nécessaires#

Maintenant que nous avons créé un nouvel utilisateur, mettons à jour le système d'exploitation et installons les paquets requis pour faire fonctionner notre noeud producteur de blocs.

apt update -y

apt upgrade -y
sudo apt-get install automake build-essential pkg-config libffi-dev libgmp-dev libssl-dev libtinfo-dev libsystemd-dev zlib1g-dev make g++ tmux git jq wget libncursesw5 libtool autoconf -y

Pare-feu et ports SSH#

Nous avons créé un utilisateur "non-root", mis à jour notre système d'exploitation et installé les packages nécessaires. Configurons maintenant notre pare-feu, ainsi que d'ouvrir les ports spécifiques que nous utiliserons pour notre serveur.

3.06 Tout d'abord, désactivons notre pare-feu, pour ne pas être verrouillé lors de sa configuration:

ufw disable

3.07 Nous allons maintenant éditer le fichier sshd_config, afin de pouvoir changer le port d'accès par défaut, ainsi que quelques paramètres supplémentaires:

nano /etc/ssh/sshd_config

3.08 Nous utiliserons le port 3100 comme port par défaut pour accéder à notre serveur. Remplacez la ligne «#Port 22» par «Port 3100». Assurez-vous de supprimer le "#":

3.09 Faites défiler vers le bas et modifions deux autres paramètres supplémentaires. Tout d'abord, remplacez "#TCPKeepAlive yes" par "TCPKeepAlive no". Deuxièmement, remplacez «#ClientAliveInterval 0» par «ClientAliveInterval 1800»:

note

Pour revenir au Terminal, enregistrez le fichier avec ctrl + o, et quittez l'éditeur avec ctrl + x.

3.10 Exécutez les commandes suivantes, qui autoriseront les connexions sortantes, mais refuseront par défaut les connexions entrantes:

ufw default deny incoming
ufw default allow outgoing

3.11 Nous allons maintenant ouvrir le port SSH que nous utiliserons pour nous connecter à notre serveur:

ufw limit proto tcp from any to any port 3100

3.12 Enfin, nous allons ouvrir le port 3000, car c'est le port que nous utiliserons pour notre noeud producteur de blocs:

ufw allow proto tcp from any to any port 3000

3.13 Maintenant, réactivez le pare-feu (assurez-vous d'entrer "y" pour confirmer):

ufw enable

3.14 Vérifions que nous avons ouvert les bons ports:

ufw status verbose

3.15 Vérifiez si votre nouvel utilisateur est dans le "sudo group":

grep '^sudo:.*$' /etc/group | cut -d: -f4

3.16 Il est temps de redémarrer le serveur:

reboot

Se connecter en tant que nouvel utilisateur#

3.17 Faites une courte pause maintenant, juste quelques minutes. Une fois que vous avez terminé, exécutez la commande suivante pour vous connecter à votre serveur, avec le nouvel utilisateur que vous avez créé un peu plus tôt dans ce guide:

ssh -i ~/.ssh/myssh YourUserName@TheIPAddressOfYourServer -p 3100
note

Assurez-vous de remplacer "YourUserName" par votre nom d'utilisateur et "TheIPAddressOfYourServer" par l'adresse IP de votre serveur.

3.18 Félicitations! Vous êtes maintenant connecté en tant que nouvel utilisateur, sur votre serveur. Supprimons maintenant l'accès de l'utilisateur root, nous serons donc sûrs à 100% que personne ne pourra prendre la position de dictateur:

sudo nano /etc/ssh/sshd_config

3.19 Remplacez la ligne "PermitRootLogin yes" par "PermitRootLogin no"

note

Pour quitter l'éditeur nano, appuyez sur ctrl + o pour enregistrer le fichier, et ctrl + x pour quitter l'éditeur.

3.20 Maintenant, rechargez le SSH daemon:

sudo service sshd reload

Et voilà! Votre serveur est maintenant enfin configuré et vous pouvez passer à la quatrième étape de ce guide.
