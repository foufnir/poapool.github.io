2. Configuration du serveur (DigitalOcean)

Dans cette deuxième partie du guide, nous installerons notre premier serveur sur DigitalOcean. Si vous utilisez un autre fournisseur de VPS, n'hésitez pas à ignorer cette étape et passez directement à la troisième partie du guide.

Générer vos clés SSH#

Il existe deux façons de se connecter à un serveur. La première consiste à utiliser un mot de passe. Ceci est généralement considéré comme non optimal, simplement car les mots de passe peuvent être piratés. Pour nous connecter à nos serveurs d'une manière parfaitement sûre, nous utiliserons donc ce que l'on appelle une connexion SSH (nous n'entrerons pas ici dans le détail mais de nombreux articles traitent du sujet). Cela signifie que nous devrons générer une paire de clés SSH, une clé publique et une privée.

2.01 Ouvrez le terminal, et exécutez la commande suivante:

cd .ssh
note

Si vous n'avez pas ce dossier .ssh, vous pouvez en créer un avec la commande suivante:

mkdir .ssh

Vous pouvez ensuite vous positionner dans ce dossier en utilisant la commande suivante:

cd .ssh

2.02 Vous êtes désormais dans le dossier .ssh, vous pouvez maintenant générer une paire de clés SSH en utilisant la commande suivante:

ssh key-gen

2.03 Tapez "myssh" comme nom de fichier.

2.04 Entrez un mot de passe, confirmez le, et vous devriez voir apparaitre cela dans votre terminal:

2.05 Verrouillons notre clé SSH privée pour plus de sécurité:

chmod 400 myssh

Félicitations! Vous avez maintenant créé avec succès votre propre paire de clés SSH, qui vous permettra de vous connecter à un serveur, que nous allons maintenant créer.

Création de votre serveur DigitalOcean#

Maintenant que nous avons nos propres clés SSH que nous utiliserons pour nous connecter au serveur, créons ce serveur. Si vous êtes nouveau sur DigitalOcean, vous devrez y déposer 20 $, dont vous aurez besoin lors de l'enregistrement de votre compte DigitalOcean.

note

Selon que vous ayez déjà utilisé DigitalOcean ou non par le passé, vous pouvez avoir un processus de configuration du serveur différent. Si tel est le cas, portez une attention particulière aux spécifications du serveur que nous utiliserons.

2.06 Ouvrez votre navigateur, accédez à digitalocean.com, créez un compte ou connectez-vous si vous en avez déjà un.

2.10 N'hésitez pas à sélectionner un emplacement de serveur différent.

note

Il y a deux choses à prendre en compte concernant l'emplacement du serveur.

Premièrement, la plupart des opérateurs de stake pool sont basés soit aux États-Unis, soit en Europe. Idéalement, vous voulez que vos nœuds y soient actifs, de sorte que la connexion aux autres nœuds du réseau soit aussi rapide que possible.

Deuxièmement, vous ne voulez pas prendre un serveur à l'autre bout du monde, car votre temps de réponse sera alors plus lente.

Connexion à votre serveur#

2.16 Pour vous connecter à votre serveur, exécutez simplement la commande suivante dans un nouvel onglet du terminal:
