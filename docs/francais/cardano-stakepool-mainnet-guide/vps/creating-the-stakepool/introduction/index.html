1. Introduction

Dans ce guide, nous allons créer une stake pool opérant entièrement dans le cloud à partir de zéro grâce à l'utilisation d'un VPS.

Qu'est-ce qu'un VPS?#

VPS signifie Virtual Private Server, ou Serveur Virtuel Privé. Vous pouvez le voir comme un ordinateur fonctionnant dans le cloud et que vous pouvez louer.

De combien de VPS ai-je besoin pour gérer une stake pool?#

Idéalement, 3 (selon les récommandations d'IOHK). Mais plus il y en a, mieux c'est, l'idée est de décentraliser autant que possible votre infrastructure, pour réduire les risques de défaillance. Pour cette raison, vous ne voulez clairement pas faire fonctionner tous vos noeuds sur le même serveur, car cela rendrait votre noeud producteur de blocs vulnérable aux attaques (nous expliquerons l'architecture plus en détail dans la suite de ce guide).

Dans un monde parfait, vous feriez fonctionner votre noeud producteur de blocs sur un serveur spécifique. Et chacun de vos noeuds relais serait sur un serveur différent, avec un fournisseur différent. Mais c'est bien sûr le cas idéal. Tout le monde ne peut pas se permettre de payer plusieurs serveurs chaque mois. Donc, ce que la plupart des opérateurs de stake pool ont, c'est un serveur spécifique pour leur noeud producteur de blocs, puis un serveur différent (parfois avec le même fournisseur) pour leurs 2 noeuds relais.

Quel fournisseur de VPS dois-je choisir?#

Il existe de nombreux fournisseurs de VPS, notamment AWS, Google Cloud, Microsoft Azure et DigitalOcean. Chez POA, nous utilisons CLOUD.co.za, un fournisseur sud-africain. Mais c'est surtout parce que nous voulons être entièrement basés en Afrique. Si vous recherchez un fournisseur VPS, nous vous recommandons DigitalOcean, car la configuration est très facile, c'est financièrement assez abordable, nous ferons l'ensemble du guide en utilisant DigitalOcean.

Cependant, notez que vous pouvez utiliser le fournisseur de votre choix, le fonctionement sera presque toujours le même.

Quel système d'exploitation dois-je utiliser sur mes VPS?#

Dans ce guide, nous utiliserons la distribution Linux Ubuntu sur nos VPS. Qu'il s'agisse d'Ubuntu 18.04 ou 20.04, cela n'a pas vraiment d'importance. Mais la version préférée est bien sûr la dernière version stable.

Quelle configuration allons-nous utiliser dans ce guide?#

Dans ce guide, nous allons configurer notre stake pool en utilisant deux VPS sur DigitalOcean. Un VPS avec notre noeud producteur de blocs, et l'autre avec nos deux noeuds relais.

Ce n'est certainement pas la configuration parfaite, mais c'est la plus cohérente économiquement parlant, ainsi qu'en termes d'expérience utilisateur. Si vous souhaitez configurer un troisième VPS pour avoir une configuration optimale, il vous suffira de refaire ce que vous avez fait pour le deuxième serveur.

Chacun de nos serveurs aura 4 Go de RAM et 80 Go de SSD. Le processeur n'a pas vraiment d'importance, la RAM en a, et plus vous avez de stockage SSD, mieux ce sera sur le long terme.
