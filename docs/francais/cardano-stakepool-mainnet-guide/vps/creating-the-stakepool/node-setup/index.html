4. Configurer notre noeud producteur de block

Dans cette quatrième partie du guide, nous allons installer le noeud à partir du code source et préparer les fichiers de configuration afin que vous puissiez commencer à le faire fonctionner.

Téléchargement et configuration de Cabal#

Pour configurer nos noeuds, nous aurons besoin de la bibliothèque Cabal, que nous utiliserons pour construire nos noeuds.

4.01 Exécutez la commande suivante pour télécharger Cabal:

wget https://downloads.haskell.org/~cabal/cabal-install-3.2.0.0/cabal-install-3.2.0.0-x86_64-unknown-linux.tar.xz

4.02 Décompressez le fichier:

tar -xf cabal-install-3.2.0.0-x86_64-unknown-linux.tar.xz

4.03 Supprimez le fichier téléchargé, car maintenant nous l'avons décompressé, nous n'en avons plus besoin:

rm cabal-install-3.2.0.0-x86_64-unknown-linux.tar.xz cabal.sig

4.04 Créez maintenant le dossier .local/bin, où nous stockerons Cabal:

mkdir -p ~/.local/bin

4.05 Déplacez Cabal vers le nouveau dossier .local/bin:

mv cabal ~/.local/bin/

4.06 Ajoutez la ligne de code (avec "export") à votre fichier .bashrc pour ajouter le dossier .local/bin à votre PATH:

nano .bashrc
export PATH="~/.local/bin:$PATH"

4.07 Laissez le fichier et actualisez-le:

note

Pour quitter l'éditeur, enregistrez le fichier avec ctrl + o, et quittez l'éditeur avec ctrl + x.

source .bashrc

4.08 Mettez à jour Cabal:

cabal update

4.09 Et enfin, vérifiez si vous avez la bonne version:

cabal --version

Téléchargement et configuration de GHC#

GHC (Glasgow Haskell Compiler), est comme Cabal, un outil qui nous aidera à construire nos noeuds.

4.10 Exécutez la commande suivante pour télécharger le fichier:

wget https://downloads.haskell.org/~ghc/8.6.5/ghc-8.6.5-x86_64-deb9-linux.tar.xz

4.11 Décompressez le fichier:

tar -xf ghc-8.6.5-x86_64-deb9-linux.tar.xz

4.12 Supprimer le fichier initialement téléchargé:

rm ghc-8.6.5-x86_64-deb9-linux.tar.xz

4.13 Entrez dans le dossier GHC:

cd ghc-8.6.5

4.14 Configurez-le:

./configure

4.15 Et installez-le:

sudo make install

Téléchargement et configuration de Libsodium#

Comme GHC et Cabal, Libsodium est un outil nécessaire pour faire fonctionner nos noeuds.

4.16 Exécutez la commande suivante pour télécharger le fichier:

git clone https://github.com/input-output-hk/libsodium

4.17 Entrez dans le dossier:

cd libsodium

4.18 Exécutez cette commande magique:

git checkout 66f017f1

4.19 Configurez le fichier:

./autogen.sh
./configure

4.20 Et installez-le:

make

sudo make install

4.21 Quittez le dossier, modifiez le fichier .bashrc et ajoutez-y les deux lignes de code (avec "export"):

cd
nano .bashrc

export LD_LIBRARY_PATH="/usr/local/lib:$LD_LIBRARY_PATH"
export PKG_CONFIG_PATH="/usr/local/lib/pkgconfig:$PKG_CONFIG_PATH"

note

Pour quitter l'éditeur, enregistrez le fichier avec ctrl + o et quittez avec ctrl + x.

Téléchargement et configuration du cardano-node#

Dans cette partie, nous allons télécharger le code source de cardano-node, et le configurer grâce à l'utilisation de Cabal.

4.22 Exécutez la commande suivante pour télécharger le fichier:

git clone https://github.com/input-output-hk/cardano-node.git

4.23 Entrez le dossier avec la commande suivante:

cd cardano-node

4.24 Récupérez toutes les versions possibles, puis listez-les:

git fetch --all --tags
git tag

4.25 Exécutez la commande suivante pour sélectionner la version que vous souhaitez utiliser:

note

Pour savoir quelle est la dernière version, allez sur cette page et voyez quelle version est en haut de la liste.

git checkout tags/TheVersionYouWouldLikeToUse

note

Comme vous pouvez le voir, nous utilisons cardano-node 1.19.0 car c'est la dernière version au moment où nous écrivons le guide. Cependant, assurez-vous d'utiliser la dernière version comme expliqué ci-dessus.

4.26 Préparez un bouquin (cette commande peut prendre beaucoup de temps) et exécutez la commande suivante:

cabal build all

4.27 Exécutez maintenant les commandes suivantes pour déplacer les exécutables cardano-node et cardano-cli vers le dossier .local/bin:

note

Assurez-vous de remplacer "TheVersionYouWouldLikeToUse" dans les commandes suivantes par la version que vous avez sélectionnée à l'étape 4.25.

cp -p dist-newstyle/build/x86_64-linux/ghc-8.6.5/cardano-node-TheVersionYouWouldLikeToUse/x/cardano-node/build/cardano-node/cardano-node ~/.local/bin/
cp -p dist-newstyle/build/x86_64-linux/ghc-8.6.5/cardano-cli-TheVersionYouWouldLikeToUse/x/cardano-cli/build/cardano-cli/cardano-cli ~/.local/bin/

4.28 Enfin, vérifiez si vous avez la bonne version:

cardano-cli --version

Bravo! Vous pouvez maintenant passer à l'étape 5 du guide, où vous allez enfin lancer les noeuds.
