6. Clés et adresses

Dans cette, très longue (et ennuyeuse! (Mais importante!!)), partie du guide, nous allons créer toutes nos clés et adresses. Que ce soit notre payment addresses, staking keys, KES keys, cold keys,... Si vous ne savez pas ce que signifie cela, pas de soucis.

note

À partir de maintenant, assurez-vous que votre noeud est toujours en cours d'exécution et entièrement synchronisé avec la blockchain. Si ce n'est pas le cas, les commandes incluant les transactions et la récupération des paramètres du réseau ne fonctionneront tout simplement pas.

Alors ouvrez d'abord un onglet dans le Terminal, connectez-vous à votre serveur et lancez-y votre noeud (ne fermez pas l'onglet!). Ensuite, ouvrez un nouvel onglet du Terminal, connectez-vous à votre serveur, et c'est là que vous exécuterez toutes les commandes.

Clés et adresses personnelles#

Tout d'abord, nous allons commencer par configurer nos clés et adresses personnelles, que nous utiliserons pour réaliser des transactions, stacker nos adas,...

6.01 Sortez du répertoire dans lequel vous vous trouvez, puis créez un dossier spécifique pour toutes vos clés dans le répertoire cardano-node, et enfin cd dedans:

cd
mkdir cardano-node/pool-keys
cd cardano-node/pool-keys

6.02 Maintenant que nous sommes dans le nouveau dossier pool-keys, vous pouvez commencer à exécuter toutes les commandes suivantes pour générer les clés nécessaires:

Paire de clés de paiement#

cardano-cli shelley address key-gen --verification-key-file payment.vkey --signing-key-file payment.skey

Paire de clés de staking#

cardano-cli shelley stake-address key-gen --verification-key-file stake.vkey --signing-key-file stake.skey

Adresse de paiement#

cardano-cli shelley address build --payment-verification-key-file payment.vkey --stake-verification-key-file stake.vkey --out-file payment.addr --mainnet

Adresse de staking#

cardano-cli shelley stake-address build --stake-verification-key-file stake.vkey --out-file stake.addr --mainnet

6.03 Vérifiez si vous disposez de tous les fichiers requis:

ls

Funding our wallet#

Afin d'enregistrer notre adresse de staking, d'enregistrer notre stake pool, ainsi que de payer tous les frais de transaction liés à ces actions, nous allons avoir besoin d'ada.

6.04 Dans le dossier pool-keys, exécutez la commande suivante:

cat payment.addr
note

Si l'adresse ne s'affiche pas bien, n'hésitez pas à exécuter la commande suivante pour l'afficher d'une autre manière:

more payment.addr

L'adresse que le terminal vous donne est votre adresse de paiement. C'est là que vous devrez envoyer les fonds.

Combien dois-je envoyer?

Au moins 550 ada. Cela couvrira l'enregistrement de la stake pool, l'enregistrement de l'adresse de stacking et tous les frais de transaction pendant un certain temps.

Cependant cela ne pas couvre le pledge. Donc, si vous souhaitez pledger à votre stake pool, vous devrez également envoyer ce montant de pledge à l'adresse stockée dans le fichier payment.addr.

6.05 Sortez du dossier pool-keys, puis éditez le fichier .bashrc et ajoutez-y le chemin d'accès au fichier node.socket:

cd
nano .bashrc
note

Assurez-vous de changer YourUSERNAME avec votre nom d'utilisateur.

export CARDANO_NODE_SOCKET_PATH=/home/YourUSERNAME/cardano-node/bp-node/db/node.socket

6.06 Quittez l'éditeur et actualisez le fichier .bashrc:

note

Pour quitter l'éditeur, enregistrez le fichier avec ctrl + o et partez avec ctrl + x.

source .bashrc

6.07 Revenons maintenant à notre dossier pool-keys, et vérifions si les fonds sont arrivés dans notre portefeuille:

cd cardano-node/pool-keys
cardano-cli shelley query utxo --address $(cat payment.addr) --mainnet

Indiqué par la flèche de gauche, vous pouvez voir votre adresse UTxO, et indiqué par la flèche de droite, le montant d'ada du portefeuille, en Lovelaces. Au milieu, vous pouvez voir le montant des transactions effectuées à l'UTxO.

note

Vous avez peut-être vu que nous n'avons déposé que 10 ada, et non les 550 ada dont nous parlions précédemment dans ce guide. C'est parce que nous n'allons pas enregistrer une nouvelle stake pool, 550 ada valent déjà plus de 50 $ au moment de la rédaction de ce guide. Cependant, pas de soucis, vous aurez vous les 550 ada que vous aurez envoyés.

Enregistrer notre adresse de staking sur la blockchain#

Nous avons maintenant créé de nombreuses adresses, mais ce dont nous avons besoin maintenant, c'est d'enregistrer notre adresse de staking, afin de rendre tout ceci officiel au yeux du réseau.

6.08 Tout d'abord, récupérons les paramètres de protocole du réseau:

cardano-cli shelley query protocol-parameters --mainnet --out-file protocol.json

6.09 Créez le certificat d'enregistrement:

cardano-cli shelley stake-address registration-certificate --stake-verification-key-file stake.vkey --out-file stake.cert

6.10 Maintenant, calculons d'abord les frais, afin que nous puissions nous assurer que la transaction soit effectuée correctement:

note

Assurez-vous de remplacer YourOwnUTxO par l'adresse UTxO que vous avez reçue à l'étape 6.07, et TheNumberOfTransactionsMadeTotheUTxO par le numéro qui se trouvait à droite de votre adresse UTxO après avoir exécuté la commande de l'étape 6.07. Laissez le hashtag entre les deux!

Par exemple, nous pourrions exécuter la commande suivante:

cardano-cli shelley transaction build-raw --tx-in e3fcd395a7252f64f7dcf2f40b91a46ef5244e0785ead38b9317562d8317ab4c#1 --tx-out $(cat payment.addr)+0 --ttl 0 --fee 0 --out-file tx.raw --certificate-file stake.cert
cardano-cli shelley transaction build-raw --tx-in YourOwnUTxO#TheNumberOfTransactionsMadeTotheUTxO --tx-out $(cat payment.addr)+0 --ttl 0 --fee 0 --out-file tx.raw --certificate-file stake.cert

cardano-cli shelley transaction calculate-min-fee --tx-body-file tx.raw --tx-in-count 1 --tx-out-count 1 --witness-count 1 --byron-witness-count 0 --mainnet --protocol-params-file protocol.json

Comme vous pouvez le voir sur la photo ci-dessus, les frais dans notre cas sont de 172453 Lovelaces (1 ada = 1 000 000 Lovelaces), soit près d'un cinquième d'ada. Cela peut et sera probablement différent pour vous. Assurez-vous donc de vérifier les frais lors de l'exécution des commandes suivantes.

6.11 Maintenant, vérifions le fichier protocol.json, pour voir combien nous devons déposer sur notre adresse de staking:

cat protocol.json

Comme vous pouvez le voir sur la photo ci-dessus, le dépôt minimum pour notre adresse de staking représente dans notre cas 2000000 Lovelaces, soit exactement 2 ADA. Cela peut et sera probablement différent pour vous. Assurez-vous donc de vérifier les frais lors de l'exécution des commandes suivantes.

6.12 C'est l'heure des maths! Calculons combien nous devons rembourser notre propre UTxO pour l'enregistrement de l'adresse de staking:

note
  • Assurez-vous de remplacer TheAmountOfLovelacesInYourUTxO par le montant en Lovelaces que vous avez dans votre portefeuille (payment.addr). Si vous ne savez plus combien cela représente, vous pouvez le découvrir avec la commande suivante:
cardano-cli shelley query utxo --address $(cat payment.addr) --mainnet

  • Remplacez TheFeeWeCalculatedInStep6Dot10 Avec les frais en Lovelaces que nous avons calculés à l'étape 6.10.

  • Enfin, remplacez TheValueForKeyDepositInYourProtocolDotJsonFile, par la valeur que vous avez trouvée dans votre fichier protocol.json lors de l'étape 6.11.

Dans notre cas, voici le calcul:

expr 10000000 - 172453 - 2000000
expr TheAmountOfLovelacesInYourUTxO - TheFeeWeCalculatedInStep6Dot10 - TheValueForKeyDepositInYourProtocolDotJsonFile

Comme vous pouvez le voir sur la photo ci-dessus, nous devrons rembourser 7827547 Lovelaces, soit près de 8 ADA à notre propre UTxO.

6.13 Encore un peu plus de maths pour calculer le TTL:

note

Le TTL, ou Time To Live, est la limite de temps que l'on autorise pour que notre transaction soit incluse dans un bloc, si elle n'est pas dans un bloc avant le slot indiqué, la transaction sera annulée.

cardano-cli shelley query tip --mainnet

Sur la photo, vous pouvez voir que j'ai indiqué un numéro de slot. Ce n'est pas le TTL. Pour calculer notre TTL, nous allons ajouter 200, à la valeur du numéro de slot. Cela laisse donc 200 secondes (soit un peu plus de 3 minutes) pour que la transaction soit incluse dans un bloc.

6.14 Exécutez la commande suivante pour calculer le TTL

note

Assurez-vous de remplacer YourSlotNumberValue par la valeur de slotNo que vous avez trouvée à l'étape 6.13.

Dans notre cas, voici le calcul:

expr 7054953 + 1000
note

À partir du moment où vous avez calculé le TTL, ne prenez pas de pause-café... car la chaîne elle n'en prend pas, et si vous prenez trop de temps pour exécuter la commande, votre TTL sera expiré et vous devrez recommencer. Si vous pensez que vous allez prendre un peu plus de temps que la personne moyenne pour exécuter les commandes suivantes, n'hésitez pas à ajouter 1000 (secondes) supplémentaires à la valeur de slot.

expr YourSlotNumberValue + 1000

6.15 Enregistrons notre adresse de staking!

note
  • Assurez-vous de remplacer YourOwnUTxO par votre adresse UTxO. Vous ne vous en souvenez pas ou ne le trouvez pas? Voici la commande:
cardano-cli shelley query utxo --address $(cat payment.addr) --mainnet

  • Remplacez TheNumberOfTransactionsMadeTotheUTxO par le nombre de transactions effectuées sur l'UTxO. Vous pouvez le trouver en exécutant la commande ci-dessus. Assurez-vous de conserver le hashtag!

  • Remplacez TheAmountToPayBackToYourUTxOWeCalculatedInStep6Dot12 par le montant que nous avons calculé à l'étape 6.12 pour rembourser notre propre UTxO.

  • Remplacez TheFeeWeCalculatedInStep6Dot10 par les frais de transaction que nous avons calculés à l'étape 6.10.

Dans notre exemple, voici la commande que nous exécutons:

cardano-cli shelley transaction build-raw --tx-in e3fcd395a7252f64f7dcf2f40b91a46ef5244e0785ead38b9317562d8317ab4c#1 --tx-out $(cat payment.addr)+7827547 --ttl 7056825 --fee 172453 --out-file tx.raw --certificate-file stake.cert
cardano-cli shelley transaction build-raw --tx-in YourOwnUTxO#TheNumberOfTransactionsMadeTotheUTxO --tx-out $(cat payment.addr)+TheAmountToPayBackToYourUTxOWeCalculatedInStep6Dot12 --ttl TheTTLWeCalculatedInStep6Dot14 --fee TheFeeWeCalculatedInStep6Dot10 --out-file tx.raw --certificate-file stake.cert

6.16 Signez la transaction:

cardano-cli shelley transaction sign --tx-body-file tx.raw --signing-key-file payment.skey --signing-key-file stake.skey --mainnet --out-file tx.signed

6.17 Et soumettez-le au réseau!

cardano-cli shelley transaction submit --tx-file tx.signed --mainnet

Bon boulot! Votre adresse de staking est maintenant officiellement enregistrée sur la chaîne!

Si vous avez un message d'erreur, il y a sûrement une coquille dans les lignes de code que vous avez utilisées pendant le processus de création de la transaction. Vérifiez vos calculs, assurez-vous que vous avez utilisées les bonnes commandes (et non les mêmes que les notres), et si vous n'arrivez vraiment pas les trouver, n'hésitez pas à nous solliciter dans le fil Telegram POA. Nous ferons en sorte d'y répondre au plus vite!

Clés de la stake pool#

Maintenant que nous avons généré et enregistré avec succès nos clés de staking personnelles, nous pouvons maintenant passer à celles de la stake pool.

6.18 Exécutez la commande suivante afin de générer vos fichiers cold.vkey, cold.skey et cold.counter:

cardano-cli shelley node key-gen --cold-verification-key-file cold.vkey --cold-signing-key-file cold.skey --operational-certificate-issue-counter-file cold.counter

6.19 Générez maintenant votre paire de clés VRF:

cardano-cli shelley node key-gen-VRF --verification-key-file vrf.vkey --signing-key-file vrf.skey

6.20 Générez votre paire de clés KES:

cardano-cli shelley node key-gen-KES --verification-key-file kes.vkey --signing-key-file kes.skey

6.21 Vérifiez si vous les avez toutes:

ls

Générer notre Operational Certificate#

6.22 Tout d'abord, exécutez la commande suivante pour vérifier le nombre de slots par KES:

note

Assurez-vous de remplacer YourUSERNAME par votre nom d'utilisateur.

cat /home/YourUSERNAME/cardano-node/bp-node/mainnet-shelley-genesis.json | grep KESPeriod

6.23 Interrogons maintenant le "mainnet tip":

cardano-cli shelley query tip --mainnet

Comme vous pouvez le voir sur l'image ci-dessus, le nombre d'emplacements par période est dans notre cas 129600, et le slotNo actuel est 7059825.

6.24 Encore une peu de Maths! Divisez la valeur de slotNo par la valeur de KESPeriod que nous avons calculée à l'étape 6.21:

note

  • Assurez-vous de remplacer TheSlotNumberYouFoundInStep6Dot22 par la valeur de slotNo que vous avez trouvée à l'étape 6.22.

  • Remplacez TheNumberOfSlotsPerPeriodYouFoundInStep6Dot21 par la valeur du nombre d'emplacements par période que vous avez trouvé à l'étape 6.21.

Voici la commande que nous devons exécuter dans notre exemple:

expr 7059825 / 129600
expr TheSlotNumberYouFoundInStep6Dot22 / TheNumberOfSlotsPerPeriodYouFoundInStep6Dot21

Comme vous pouvez le voir sur l'image ci-dessus, le période KES est de 54 dans notre cas. Ce sera probablement différent pour vous. Assurez-vous donc de vérifier cela lors de l'exécution de la commande suivante.

6.25 Générer le certificat

note

Assurez-vous de remplacer TheNumberYouFoundForTheKESPeriodInStep6Dot24 par la valeur que vous avez trouvée pour la période KES à l'étape 6.24.

Voici la commande que nous devons exécuter:

cardano-cli shelley node issue-op-cert --kes-verification-key-file kes.vkey --cold-signing-key-file cold.skey --operational-certificate-issue-counter cold.counter --kes-period 54 --out-file node.cert
cardano-cli shelley node issue-op-cert --kes-verification-key-file kes.vkey --cold-signing-key-file cold.skey --operational-certificate-issue-counter cold.counter --kes-period TheNumberYouFoundForTheKESPeriodInStep6Dot24 --out-file node.cert
